# Specification - optimize_usage_docs_20260108

## Overview
先のトラックで達成した劇的なパフォーマンス向上（CPU負荷 85% 削減）の成果を `README.md` に反映する。開発者がこのドライバの性能を最大限に引き出すための「最適化ガイド」を具体的なスニペットと共に掲載し、さらに詳細な内部アルゴリズムの解説を「付録」として追加することで、プロジェクトの技術的信頼性と有用性をアピールする。

## Functional Requirements
1.  **実績の誇示**:
    - `pigpiod` の負荷が 64% から 9% まで低減した定量的データを、一目でわかる形式（表など）で掲載。
2.  **開発者向け「パフォーマンス最適化ガイド」**:
    - **Dirty Rectangle（動的差分更新）の最大活用**:
        - `display(image)` に「変化した部分だけを描画したイメージ」を渡す実装パターンをコードスニペットで紹介。
    - **背景・静的パーツのキャッシュ戦略**:
        - `PIL.Image` の `copy()` と `paste()` を組み合わせて、不変な背景を再利用する具体的なコード例。
    - **FPS（更新頻度）の適切な制御**:
        - `time.sleep(1.0 / fps)` 等を用いた、視覚的滑らかさと負荷のトレードオフの管理。
3.  **付録：内部アルゴリズムの解説**:
    - **動的差分抽出**: `ImageChops.difference` と `getbbox()` を用いた高速な領域特定手法。
    - **低レベル通信最適化**: DC/CSピンの状態キャッシュによるシステムコール（GPIO操作）の最小化。
    - **ウィンドウ設定の最適化**: 冗長な `CASET`/`RASET` 送信の排除。

## Non-Functional Requirements
- **アクセシビリティ**: 開発者が「自分のコードをどう直せばいいか」がすぐわかるように、解説よりもコード例を優先する。
- **ドキュメント構造**: README の主要部分は利用方法に絞り、アルゴリズムの詳細は末尾の「付録」に分離して可読性を保つ。

## Acceptance Criteria
- `README.md` に「パフォーマンス実績」「最適化ガイド（スニペット付き）」「内部アルゴリズム解説（付録）」が追加されていること。
- 推奨されるコードパターンが、最新の `roboface3.py` の実装と整合していること。

## Out of Scope

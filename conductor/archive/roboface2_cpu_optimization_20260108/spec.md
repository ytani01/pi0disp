# Specification: `samples/roboface2.py` のCPU負荷削減

## 1. 概要
Raspberry Pi Zero 等の低リソース環境において、`samples/roboface2.py` が過剰なCPUリソースを消費する問題を改善する。描画の最適化とメインループの制御を導入し、アニメーションの品質を維持しつつシステム負荷を軽減する。

## 2. 機能要件
- **メインループのウェイト処理の適正化**: 
    - 現在のビジーウェイトに近いループ処理を改善し、適切な `sleep` またはタイマー制御を導入してCPUの空回りを防ぐ。
- **描画・送信スキップロジックの実装**:
    - 表情の状態（`RfState`）に変化がない場合、PILによる画像生成およびSPIによるディスプレイ送信をスキップする「Dirtyフラグ」的な仕組みを導入する。
- **更新頻度（FPS）の調整**:
    - 設定ファイル（`RfConfig`）などでFPSを調整可能にし、視覚的な滑らかさを損なわない範囲で更新頻度を最適化する（デフォルト値の見直しを含む）。

## 3. 非機能要件
- **パフォーマンス目標**: `top` または `htop` コマンドで確認できる `%CPU` が、現状と比較して有意に低下すること。
- **品質維持**: アニメーションの応答性や滑らかさを極端に損なわないこと。

## 4. 受け入れ条件
- `samples/roboface2.py` を実行した際、CPU使用率が改善されていることが確認できること。
- ランダムモードおよび対話モードにおいて、アニメーションが期待通り動作し、目立った遅延やカクつきが発生しないこと。
- `top`/`htop` で数値的な改善（例：アイドル時や静止時の負荷大幅減）が確認できること。

## 5. スコープ外
- レンダリングエンジン（PIL）自体の根本的な高速化（C拡張の導入など）。
- ハードウェア側の変更。

# Track Specification: Refactor lcd_check command

## Overview
`src/pi0disp/commands/lcd_check.py` をリファクタリングし、**シンプルでスッキリとしたクリーンなコード**に刷新します。巨大な関数を分割し、UI・ロジック・設定管理の責務を明確に分けることで、可読性とメンテナンス性を劇的に向上させます。

## Functional Requirements
- **責務の分離 (Simple & Clean)**: 
    - 複雑な条件分岐や巨大な関数を、目的別の小さな関数やクラスへ整理。
    - ウィザードの「進行」と「各ステップのロジック」を分離。
- **プラグインライクな拡張性**: 
    - 新しいチェック項目を追加する際、既存の巨大な `if-else` を編集するのではなく、シンプルなリストや辞書への登録で済むような見通しの良い構造。
- **インターフェースの整理**:
    - ディスプレイへの描画部分を整理し、コードの重複を排除。
- **設定保存の簡素化**: `pi0disp.toml` への書き込み処理を分かりやすく整理。

## Non-Functional Requirements
- **直感的な可読性**: 誰が見ても「どこで何をしているか」が即座に理解できるコードスタイル。
- **テストの容易性**: ロジックが分離されているため、各ステップの動作を個別にテスト可能。
- **既存機能の維持**: ウィザードの挙動やユーザー体験は維持。
- **実機検証と事実に基づく開発**: 憶測を排除し、デバッグログやキャプチャで動作を確実に検証する。

## Acceptance Criteria
- [ ] `lcd_check.py` のメインロジックが簡潔（例：数十行程度）になり、詳細がサブコンポーネントに委譲されている。
- [ ] 重複コードが排除され、一貫性のあるシンプルな実装になっている。
- [ ] `pytest` でウィザードの各ステップを個別に検証可能。
- [ ] 既存の `lcd-check` コマンドの動作が従来通り正常である。
- [ ] `mise run lint` をパスしている。

## Out of Scope
- 過度な抽象化やオーバーエンジニアリング。
- 今回のトラックでの新規チェック項目の実装。
